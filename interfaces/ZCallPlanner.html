order

  • order(first: ZTask, second: ZTask): void
    • Defined in run-z/run-z/src/core/plan/call-planner.ts:67

    Establishes the task execution order.

    The call to this method does not cause any of the tasks to be executed.

    When any of the tasks executed it first executes its prerequisites. I.e. the tasks ordered before it. The task itself will be executed only after each prerequisite completes, unless that prerequisite can be executed in parallel.

    Contradictory execution order causes one of the tasks to be executed before prerequisite.

    Parameters

    • first: ZTask

      The task executed first.

    • second: ZTask

      The task executed after the first one.

    Returns void
