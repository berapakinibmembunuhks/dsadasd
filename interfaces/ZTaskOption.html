    Reads up to the maximum number of the remaining command line arguments and treat them as option values.

    Reads [values] and all command line arguments following them up to the end of command line or their maximum number.

values

  • values(max?: undefined | number): keyof string[]

  • Inherited from ZTaskOption.values

    • Defined in run-z/optionz/d.ts/option.d.ts:59

    Reads up to the maximum number of values of the option.

    Values are command line options following the currently processed one. Their number depend on the {@link ZOptionSyntax option syntax}. E.g. a --name=value syntax supports one value, while --name value1 value2 ... syntax supports any number of them.

    Calling this method marks the option and value arguments as recognized. This can be changed by calling any recognition method again.

    When called on already recognized option this method just returns the values previously returned to the reader that recognized them.

    Parameters

    • Optional max: undefined | number

      The maximum number of values to read.

    Returns keyof string[]

    Option values array.
